# Baseline Spec — Definition of Done

## Criteria

The Baseline stage is complete when **all** of the following are true:

| # | Criterion | Description |
|---|-----------|-------------|
| 1 | **Problem is specific and validated** | The core problem is written in concrete terms, not vague generalities. It describes who has the problem, what the problem is, and why it matters. |
| 2 | **Target users are clearly defined** | At least 2 user personas are described with roles, context, pain points, and goals. |
| 3 | **Current journey is mapped** | The document describes how users currently solve (or fail to solve) the problem without the proposed product. |
| 4 | **Solution direction is clear** | The proposed solution approach is described at a conceptual level — what the product will do, not how it will be built. |
| 5 | **Happy flow is complete end-to-end** | The happy flow describes a full user journey from trigger to outcome, covering all critical touchpoints. |
| 6 | **Capabilities are system-level** | The capability list describes what the system can do (not UI components). Each capability maps to a user need. |
| 7 | **No premature feature detail** | The spec avoids detailed feature specifications, UI layouts, or technical architecture. It stays at the conceptual level. |
| 8 | **Research questions target gaps** | Research questions identify specific unknowns that must be validated before defining features. They are actionable, not rhetorical. |
| 9 | **Research prompts are ready to use** | Generated research prompts can be directly pasted into GPT Deep Research, Perplexity, or Gemini with no editing required. |
| 10 | **All 5 deliverables exist** | baseline-spec.md, capability-list.md, happy-flow.md, research-questions.md, and research-prompts.md are all created. |

## Evaluation

When evaluating the DoD:
- Mark each criterion as **PASS** or **FAIL**.
- For any FAIL, explain specifically what is missing or insufficient.
- The stage can only be marked `completed` when all 10 criteria pass.
- If criteria 1–9 pass but deliverables are missing (criterion 10), generate the missing deliverables.
